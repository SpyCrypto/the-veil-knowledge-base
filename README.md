## How to contribute
We welcome contributions of all kinds: research notes, frameworks, multimedia, community resources, and editorial improvements.

1. Read [STRUCTURE.md](STRUCTURE.md) to understand where to add content.
2. Check open issues and discussions to avoid duplicating work.
3. Fork the repository and open a PR with a clear description of your changes.
4. Use meaningful filenames and front-matter where applicable (see STRUCTURE.md for examples).

If you're unsure where something belongs, open an issue or start a discussion in the repository.
