# Frameworks

This directory contains practical frameworks, architectural models, and tools for applying research to real-world problems.

## Framework Categories

### Systems Architecture
Distributed systems design, protocol specifications, and technical architectures.

### Security Models
Threat models, access control frameworks, and security-by-design approaches.

### Governance Patterns
Organizational structures, decision-making models, and coordination mechanisms.

## Framework Structure

Each framework should include:

- **README.md** — Overview, use cases, and quick-start guide
- **architecture.md** — Design decisions, components, and how they interact
- **examples/** — Worked examples or case studies applying the framework
- **diagrams/** — Visual representations (ASCII art or images)

## Contributing a Framework

1. Create a subdirectory: `frameworks/<framework-name>/`
2. Add a `README.md` with an overview and intended audience
3. Document the framework's components and architecture
4. Provide at least one worked example
5. Include diagrams or visual aids where helpful
6. Open a PR for review
